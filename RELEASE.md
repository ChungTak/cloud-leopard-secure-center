# Release Process

The repository version is defined once in the workspace `Cargo.toml` under `[workspace.package].version`.

## Cutting a release

1. Ensure all Phase 1 plan files in `dev-docs/001_vibe_coding_plan/` are marked complete.
2. Run the release gate locally or in CI:
   ```bash
   DATABASE_URL=postgres://postgres:postgres@localhost:5432/clsc \
     scripts/release-gate.sh 0.1.0
   ```
3. Review `dist/openapi-0.1.0.diff` for breaking changes.
4. Tag the release:
   ```bash
   git tag -a v0.1.0 -m "Release 0.1.0"
   git push origin v0.1.0
   ```
5. Create a GitHub Release from the tag and attach artifacts from `dist/`.

## Artifacts

| Artifact | Location | Tool |
|---|---|---|
| Container image | `ghcr.io/chungtak/cloud-leopard-secure-center:<version>` | `docker build` |
| SBOM | `dist/sbom-<version>.json` | `syft` (CycloneDX JSON) |
| OpenAPI diff | `dist/openapi-<version>.diff` | `UPDATE_OPENAPI=1 cargo test -p http-api openapi_snapshot` |
| Release notes | `dist/release-<version>.md` | generated by `scripts/release-gate.sh` |

## Upgrade / rollback

- **Before upgrade:** back up PostgreSQL and any external NATS JetStream streams.
- **Upgrade:** run `cargo run -p migration-cli -- migrate run`, then deploy the new container image digest.
- **Rollback:** restore the database, re-deploy the previous image digest, and verify tenant cache consistency.

## UNSUPPORTED capabilities in 0.1.0

The following runtime capabilities are intentionally `UNSUPPORTED` in 0.1.0 and return stable errors:

- Upstream signaling REST/SSE transport (`RestSignalingAdapter` returns `Unavailable` when unconfigured, `Unsupported` otherwise).
- Full signaling reconciliation (`SignalingReconciler::reconcile`).
- Playback entitlement upstream media session creation (`UnsupportedMediaPort`).
- Browser player WebAssembly/codec runtime (`SecurityPlayer` renders `unsupported` fallback).
