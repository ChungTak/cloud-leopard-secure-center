# 任务完成报告规范

每个已完成任务必须创建 `reports/<task-id-lowercase>.md`。报告是验收证据，不是开发日志。

## 模板

```markdown
# TASK-ID 完成报告

## 基线
- 开始 commit：
- 完成 commit：
- 前置任务：

## 交付
- 修改的契约/行为：
- 兼容性与 migration：
- 未改变的边界：

## 验证
| 命令 | 结果 | 耗时 |
| --- | --- | --- |

## 失败与恢复场景
- 场景：结果

## 遗留事项
- 无；或引用已登记的后续任务 ID
```

## 规则

- 不粘贴无限制完整日志；保留命令、退出码和关键摘要。
- 长期性能、互操作和混沌测试可链接仓库内产物或 CI run ID。
- 跳过测试必须说明成本、环境要求和负责的后续任务；必需测试不能以 skip 完成。
- 报告不得包含 secret、内部生产地址或未脱敏数据。
- 报告中的 commit 必须能包含任务的代码、测试和相关文档。
