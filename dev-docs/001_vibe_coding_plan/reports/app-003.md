# APP-003 Inbox 与后台任务报告

## 任务目标

实现 Inbox 去重与后台 Job 的租约、重试、取消语义。

## 变更摘要

- `storage-api/src/lib.rs`：`InboxMessage`、`InboxRepository`、`InboxStatus`；`Job`、`JobRepository`、`JobStatus`。
- `storage-postgres/src/inbox_repository.rs`：`receive` 使用 `ON CONFLICT DO NOTHING` 按 `(tenant_id, consumer_id, message_id)` 去重并返回首次结果。
- `storage-postgres/src/job_repository.rs`：`schedule`/`claim`/`complete`/`fail`/`cancel`，使用 `attempts < max_attempts` 与租约时间窗口；`foundation::retry::RetryPolicy` 提供退避 + jitter + deadline。

## 验证

- `cargo clippy -p storage-postgres --all-targets -- -D warnings` 通过。
- 集成测试覆盖 ack 丢失、consumer restart、poison、lease 过期、取消、时钟推进。

## 未完成/降级

- 真实后台 worker 未接入 `cluster-adapter` 调度器；`RetryPolicy` 未与 `Job` 持久化错误分类联动，暂由调用方计算 `next_run`。
