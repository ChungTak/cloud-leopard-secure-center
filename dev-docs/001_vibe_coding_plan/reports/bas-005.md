# BAS-005 CI 与依赖供应链报告

## 目标
建立 PR/nightly CI 和依赖供应链门禁，确保 fmt/clippy/nextest/frontend/许可证/安全扫描可复现。

## 变更

- `.github/workflows/ci.yml`：PR 工作流包含
  - `rust-fmt`：`cargo fmt --all -- --check`
  - `rust-clippy`：`cargo clippy --workspace --all-targets -- -D warnings`
  - `rust-nextest`：`cargo nextest run --workspace`
  - `rust-aarch64`：`cargo check --workspace --target aarch64-unknown-linux-gnu`
  - `rust-deny` / `rust-deny`：`cargo deny check`（通过 `cargo-deny-action`）
  - `rust-lint-fixture`：验证 `fixtures/lint-violation` 因 `unwrap()` 按预期失败
  - `migration-smoke`：运行 `migration-cli` 并检查健康输出
  - `web-check`：pnpm 安装后运行 `typecheck`/`test`/`build`/`lint`/`format`
  - `secret-scan`：`trufflesecurity/trufflehog@main` 扫描提交中的密钥
- `deny.toml`：声明目标平台、允许的许可证（MIT/Apache-2.0）、advisories v2，启用 `allow-wildcard-paths` 以允许 workspace path 依赖。
- `.github/renovate.json`：Renovate 配置，major 版本不自动合并，依赖以 pin 方式提出独立 PR。
- 所有 crate `Cargo.toml` 增加 `publish = false`，使 `cargo-deny` 将 workspace 内 crate 视为私有，避免 path 依赖被误判为 wildcard。
- 为 `foundation` 添加最小单元测试，使 `cargo nextest` 非空。

## 验证命令

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo nextest run --workspace
cargo deny check
cargo check --workspace --target aarch64-unknown-linux-gnu
cargo run -p migration-cli | grep '"status":"ready"'
pnpm --dir web install --frozen-lockfile
pnpm --dir web typecheck && pnpm --dir web test && pnpm --dir web build && pnpm --dir web lint && pnpm --dir web format
```

## 验证结果

全部通过。
