# BAS-002 任务状态检查器报告

## 目标
建立可运行的任务状态检查器，确保 `001_vibe_coding_plan` 中任务 ID 唯一、前置存在、依赖无环、已完成任务有报告，并检测断链、未闭合代码围栏和占位表达式。

## 实现

- 新建 `tools/plan-check/` Rust crate，使用 `anyhow`、`regex`、`walkdir`。
- 解析 Markdown 任务章节（`### <ID>`）和前置行（`**前置：**`）。
- 支持多段字母任务 ID，如 `UPR-MED-001`、`UPR-SIG-001`。
- 检查：
  1. 任务 ID 唯一；
  2. 前置任务 ID 存在；
  3. 前置图无环；
  4. 全部勾选的任务存在 `reports/<id-lowercase>.md`；
  5. Markdown 相对链接可解析；
  6. 代码围栏成对闭合；
  7. 正文（非代码段）中不含 `todo!()`、`unimplemented!()`、`fixme`、`xxx`、`hack`、`stub` 等占位表达。
- 内置单元测试覆盖成功与故意失败 fixture（缺失前置、循环依赖、缺报告、断链、未闭合围栏、占位表达式）。

## 验证命令

```bash
cd tools/plan-check && cargo test
# 或
cargo run --manifest-path tools/plan-check/Cargo.toml -- dev-docs/001_vibe_coding_plan
```

## 验证结果

```text
$ cargo run --manifest-path tools/plan-check/Cargo.toml -- dev-docs/001_vibe_coding_plan
plan-check passed: 80 tasks, 28 files
```

```text
$ cargo test --manifest-path tools/plan-check/Cargo.toml
test result: ok. 7 passed; 0 failed
```

## 发现的基线问题

- 解析器初版无法识别 `UPR-MED-001` 这类多段任务 ID，导致误报 `MED-001` 缺失；已修正 ID 正则。
- 默认占位词 `placeholder` 与计划文档本身使用的英文术语冲突，已从默认占位词列表中移除。
