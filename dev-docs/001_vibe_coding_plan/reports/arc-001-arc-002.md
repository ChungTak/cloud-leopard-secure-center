# ARC-001 / ARC-002 架构 crate 图与自动检查报告

## 目标
验证并锁定 `apps -> adapters -> application -> domain -> foundation` 的依赖方向，建立一个可自动检查架构违规的工具。

## 变更

- `tools/architecture-test`：新增独立二进制/测试 crate，通过 `cargo metadata` 解析 workspace crate 图并验证：
  - 层级映射：`foundation`/`observability`（0）、`domain-*`（1）、`*-api` 端口（2）、`application`（3）、`http-api`/`storage-postgres`/`message-local` 适配器（4）、`security-*`/`migration-*` 应用（5）。
  - domain crate 禁止依赖 `tokio`、`axum`、`tonic`、`sqlx`、`nats`、`reqwest`、`hyper`、`prost`、`actix-web`。
  - domain 源码禁止出现 `tokio::`、`axum`、`tonic`、`sqlx` 等框架字符串。
  - `application` 源码禁止引用 `http_api`、`storage_postgres`、`message_local` 等适配器。
  - `apps` 源码禁止出现 `sqlx`、`SELECT/INSERT/UPDATE/DELETE`、`domain_` 等业务/SQL 字符串。
  - 通过临时 fixture 验证 domain -> port 层级、domain 使用框架、`apps` 写 SQL 都会失败。
- `dev-docs/001_vibe_coding_plan/03_architecture_crate_graph.md`：将 ARC-001/ARC-002 标记完成。

## 验证命令

```bash
cargo test --manifest-path tools/architecture-test/Cargo.toml
cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo nextest run --workspace
cargo deny check
cargo check --workspace --target aarch64-unknown-linux-gnu
```
