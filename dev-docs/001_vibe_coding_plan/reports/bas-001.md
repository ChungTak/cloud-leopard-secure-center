## 设计决策不可覆盖清单

1. Rust 1.96.1+、Edition 2024、resolver 3；依赖由 lockfile 固定。
2. 后端分层：`apps -> transport adapters -> application -> domain -> foundation`。
3. domain 禁止依赖 Tokio、Axum、Tonic、SQLx、NATS、HTTP/Proto DTO。
4. 数据唯一权威所有者，租户表强制启用 PostgreSQL RLS。
5. secret 只通过 `SecretProvider::resolve(SecretRef)` 解析，禁止 Debug/Serialize 并尽快 zeroize。
6. 所有外部调用需同时定义 deadline、取消、错误映射、幂等和遥测。
7. 端口优先（ports）与 API 优先（API 契约先于实现）。
8. 禁止生产代码中出现 `todo!()`、`unimplemented!()`、空 provider、固定成功返回、吞错或无界资源。

## 外部依赖可用性

- crates.io / npm 网络未单独测试；当前环境可访问 GitHub 完成仓库克隆。
- 容器镜像缓存：本机 Docker 可用，但 PostgreSQL/NATS 镜像未预拉取。
- 缺失工具记录为后续环境配置输入；当前不阻塞基线记录任务。
