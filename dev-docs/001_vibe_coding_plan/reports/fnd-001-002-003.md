# FND-001 / FND-002 / FND-003 基础类型、错误、配置报告

## 目标
为平台提供强类型标识、统一错误、请求上下文和分层配置，奠定 domain 无运行时依赖的基础。

## 变更

- `crates/foundation` 新增/扩展：
  - UUIDv7 newtypes：`TenantId`、`UserId`、`RoleId`、`OrganizationId`、`AreaId`、`DeviceId`、`CameraId`、`BindingId`、`AuditId`、`MessageId`、`NodeId`。
  - 值类型：`Revision`、`UtcTimestamp`、`Deadline`、`ResourceRef`、`PageCursor`。
  - 可注入抽象：`Clock`（`SystemClock`/`FakeClock`）、`RandomSource`（`SystemRandom`/`FakeRandom`）、`IdGenerator`（`StandardIdGenerator`/`SystemIdGenerator`）。
  - 错误模型：`PlatformError` 覆盖 14 个稳定分类，`ErrorCode` 枚举，公开 message 安全，source 不进入序列化。
  - 请求上下文：`RequestContext` 携带 request/correlation/trace/actor/tenant/organization/deadline，不含数据库连接或 HTTP extractor。
  - 配置模块 `foundation::config`：
    - `AppConfig` 合并默认值 → TOML 文件 → `CLSC_OVERRIDES` 环境变量字符串。
    - 所有配置段启用 `#[serde(deny_unknown_fields)]`，未知字段拒绝启动。
    - `Port`、`WorkerCount` 带上下限验证并通过 `TryFrom` 在反序列化时生效。
    - `SecretRef`、`SecretValue`、`SecretPort`、`EnvSecretProvider`、`StaticSecretProvider`；`SecretValue` 使用 `zeroize` 在 drop 时擦除，不实现 `Serialize`/`Debug` 暴露。
  - 根目录 `config.example.toml`：全占位值，无生产默认。
- `Cargo.toml`/`deny.toml`：添加 `chrono`、`getrandom`、`serde`、`thiserror`、`toml`、`uuid`、`zeroize`、`serde_json`（dev）并允许 `Unicode-3.0` 许可证。
- `04_foundation_types_errors_config.md`：三个任务标记完成。
