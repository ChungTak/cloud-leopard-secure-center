# APP-002 Idempotency 与 Outbox 报告

## 任务目标

实现幂等键去重与首次响应保存，确保聚合、审计、outbox 在 UoW 中原子提交。

## 变更摘要

- `storage-api/src/lib.rs`：`IdempotencyRecord`、`IdempotencyStatus`、`IdempotencyRepository` 新增 `save_or_conflict`（默认实现 + `ErrorCode::Conflict`）。
- `storage-postgres/src/idempotency_repository.rs`：覆盖 `save_or_conflict`，通过 `SELECT ... FOR UPDATE` 原子检查 digest；相同 key 不同 digest 返回 `Conflict`。
- `storage-api`/`storage-postgres`：`UnitOfWork`、`OutboxRepository` 与 `PostgresUnitOfWork` 实现聚合/审计/outbox 原子提交；outbox publisher 使用 `claim`/`mark_published` 与 batch size 限制。

## 验证

- `cargo clippy -p storage-api -p storage-postgres --all-targets -- -D warnings` 通过。
- 集成测试覆盖 100 次重复、commit/rollback 原子性、双 publisher 仅一次成功。

## 未完成/降级

- 真实 HTTP 层尚未调用 `save_or_conflict`；HTTP idempotency 中间件在 `http-api` 已有内存版本，接入 Postgres 版本需真实数据库环境。
