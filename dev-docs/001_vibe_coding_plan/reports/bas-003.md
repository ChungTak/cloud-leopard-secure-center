# BAS-003 Rust workspace 初始化报告

## 目标
建立可重复构建的 Rust monorepo 和最小可执行服务，不加入业务假实现。

## 变更

- 根 `Cargo.toml`：固定 Rust edition 2024、resolver 3，声明 workspace members、统一 `workspace.package`（版本、MSRV 1.96.1、license、repository）和 workspace lints：`unsafe_code=forbid`，`clippy::unwrap_used`、`clippy::expect_used`、`clippy::await_holding_lock=deny`。
- `rust-toolchain.toml`：固定 `1.96.1`，启用 `rustfmt`、`clippy`，并声明 `x86_64` 与 `aarch64` target。
- `LICENSE`：声明 MIT OR Apache-2.0 双许可。
- `apps/security-platform`：配置加载入口、生命周期、健康检查 JSON 输出，无领域逻辑。
- `apps/migration-cli`：配置加载入口，装配 `storage-postgres`、`message-local`、`application`、`foundation` 并健康输出。
- Phase 0 crates：
  - `crates/foundation`
  - `crates/domain-identity`、`domain-organization`、`domain-authorization`、`domain-resource`、`domain-audit`、`domain-configuration`
  - `crates/application`
  - `crates/storage-api`、`crates/storage-postgres`
  - `crates/message-api`、`crates/message-local`
  - `crates/http-api`、`crates/observability`
- 每个 crate 包含 `README.md` 说明职责与允许/禁止依赖；`Cargo.toml` 通过 `[lints] workspace = true` 继承全局 lint。
- `fixtures/lint-violation/`：独立 workspace 的 fixture，故意包含 `unwrap()` 以验证 `clippy::unwrap_used` 能在 CI 中失败。

## 验证命令

```bash
cargo fmt --all -- --check
cargo clippy --workspace --all-targets -- -D warnings
cargo test --workspace
cargo check --workspace --target aarch64-unknown-linux-gnu
cargo clippy --manifest-path fixtures/lint-violation/Cargo.toml -- -D clippy::unwrap_used  # 应失败
```

## 验证结果

全部通过；lint fixture 按预期失败。

- `cargo nextest` 与 `cargo deny` 尚未安装，将在 BAS-005/CI 任务或环境配置中补齐。
