# BAS-004 Web workspace 初始化报告

## 目标
初始化前端 monorepo，固定 Node/pnpm/React/TS/Vite/Semi/Vitest/Playwright 工具链，产出可 typecheck/test/build 的最小闭环。

## 变更

- `web/package.json`：声明 workspace root、固定 `typescript@7.0.2`、`vite@8.1.4`、`vitest@4.1.10`、`prettier@3.5.3`、`eslint@9.20.1`，`packageManager: pnpm@11.15.1`（11.12.0 为 broken release，pnpm 安装器已提示不可用）。
- `web/pnpm-workspace.yaml`：包含 `apps/*` 与 `packages/*`。
- `web/tsconfig.json`：strict、bundler moduleResolution、react-jsx、noEmit 基线。
- `web/prettier.config.mjs` 与 `web/eslint.config.mjs`：Prettier 使用单引号/尾逗号；ESLint 配置为仅检查 `.js/.mjs/.cjs` 文件，`.ts/.tsx` 由 `tsc` 负责（`typescript-eslint` 当前版本不支持 TS 7.0.2）。
- `web/apps/console`：Vite React TS 应用，装配 `@clsc/ui`、`@clsc/player`、`@clsc/api-client`，含 `App.test.tsx`。
- `web/packages/api-client`：最小 typed API client，含 vitest 测试。
- `web/packages/ui`：React 组件库（含 `Button`），使用 `@testing-library/react` + `jsdom`。
- `web/packages/player`：React 播放器组件。
- 产物使用相对 asset manifest；未从外部第三方域加载代码。

## 验证命令

```bash
pnpm --dir web install --frozen-lockfile
pnpm --dir web typecheck
pnpm --dir web test
pnpm --dir web build
pnpm --dir web lint
pnpm --dir web format
```

## 验证结果

全部通过（typecheck/test/build/lint/format）。

## 偏差说明

- `pnpm` 固定为 `11.15.1`；`11.12.0` 在 `pnpm env use` 与 `self-update` 时被明确报告为 broken release。
- ESLint 未启用 `typescript-eslint` 解析 `.ts/.tsx`，因为 `typescript-eslint@8.65.0` 明确报错不支持 TS 7.0；待上游兼容后再启用。
